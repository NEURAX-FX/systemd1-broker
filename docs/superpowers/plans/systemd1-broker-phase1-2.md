# systemd1 Broker Phase 1/2 Implementation Plan

## Objective

Build the smallest usable `org.freedesktop.systemd1` compatibility broker slice
on top of the current in-memory Manager/Unit/Job model:

1. Encode Manager `ListUnits()` and `ListJobs()` data with exact systemd tuple
   shapes.
2. Expose read-only Manager and Unit properties through an sd-bus facade.
3. Expose minimal Job lifecycle methods needed by `systemctl --wait`.
4. Add a broker executable that can serve a controlled in-process/static backend
   over a Unix socket for smoke testing.

This plan intentionally leaves full unit-file indexing, install-state handling,
policy, and a real backend adapter for later work.

## Current State

Existing files in this worktree:

1. `docs/SYSTEMD1_BROKER_SPEC.md` defines the compatibility surface and phased
   scope.
2. `src/systemd1-broker/systemd1-broker.h` declares Manager, Unit, Job, UnitInfo,
   and JobInfo helpers.
3. `src/systemd1-broker/systemd1-broker.c` implements unit registration, unit
   state mapping, job creation, job running/completion, and list-view structs.
4. `src/test/test-systemd1-broker.c` covers current state-model behavior.
5. `src/systemd1-broker/meson.build`, `src/test/meson.build`, and top-level
   `meson.build` wire the helper code and focused test.

Known verification command:

```sh
meson test -C build -v test-systemd1-broker
```

If the worktree has no build directory, configure once with:

```sh
meson setup build
```

If unrelated OpenSSL build issues appear, keep using the existing workaround:

```sh
meson configure build -Dopenssl=disabled
```

## Design Constraints

1. Keep the state model backend-agnostic; no real service manager is introduced
   in this plan.
2. Keep D-Bus encoding isolated from the plain state model so unit tests can
   exercise both separately.
3. Use exact systemd signatures from `docs/SYSTEMD1_BROKER_SPEC.md`:
   `ListUnits()` returns `a(ssssssouso)` and `ListJobs()` returns `a(usssoo)`.
4. Do not fabricate advanced systemd behavior. Unsupported methods should return
   deterministic D-Bus errors once the facade exists.
5. Prefer small helper functions over a large daemon implementation until tuple,
   property, and job semantics are covered by tests.

## Milestone 1: D-Bus Tuple Encoding Helpers

Goal: prove that the current `Systemd1BrokerUnitInfo` and
`Systemd1BrokerJobInfo` structs can be serialized with the exact ABI shapes
expected by `systemctl`.

Files to change:

1. Add `src/systemd1-broker/systemd1-broker-bus.h`.
2. Add `src/systemd1-broker/systemd1-broker-bus.c`.
3. Update `src/systemd1-broker/meson.build` to include the new source.
4. Extend `src/test/test-systemd1-broker.c` or add
   `src/test/test-systemd1-broker-bus.c` if the test grows too large.

Tests first:

1. Create an `sd_bus_message` with `sd_bus_message_new()` as in
   `src/test/test-bus-unit-util.c`.
2. Append one `Systemd1BrokerUnitInfo` into an array container using a helper
   such as `systemd1_broker_bus_append_unit_info()`.
3. Rewind/read the message and assert the tuple fields match
   `a(ssssssouso)` order: id, description, load state, active state, sub state,
   following, unit path, job id, job type, job path.
4. Repeat for one `Systemd1BrokerJobInfo` with `a(usssoo)` order: job id, unit
   id, job type, state, job path, unit path.
5. Add negative tests for missing required string fields returning `-EINVAL`.

Implementation notes:

1. Keep these helpers pure append/read-adjacent helpers; do not add Manager
   method handlers yet.
2. Return negative errno-style errors.
3. Use systemd object paths already generated by `unit_dbus_path_from_name()` and
   `systemd1_broker_job_path()`.

Verification:

```sh
meson test -C build -v test-systemd1-broker
```

## Milestone 2: Property View Helpers

Goal: make Unit and Service property data explicit before exposing it through
`org.freedesktop.DBus.Properties`.

Files to change:

1. Extend `src/systemd1-broker/systemd1-broker.h` with property view structs for
   Unit and minimal Service properties.
2. Extend `src/systemd1-broker/systemd1-broker.c` with getters that populate the
   property view structs.
3. Extend focused tests.

Tests first:

1. Add a test that `systemd1_broker_unit_get_properties()` returns correct `Id`,
   `Description`, `LoadState`, `ActiveState`, `SubState`, `FragmentPath`,
   `SourcePath`, `DropInPaths`, `UnitFileState`, `NeedDaemonReload`, `Job`, and
   `InvocationID` neutral values.
2. Add a test that `.service` units produce minimal Service properties:
   `MainPID=0`, `ExecMainPID=0`, `ControlPID=0`, `Result="success"` or empty,
   empty status strings, zero timestamps, and zero exit code/status.
3. Add a test that non-service units do not expose Service properties.

Implementation notes:

1. Start with neutral values only; do not parse unit files yet.
2. Store no `char **` ownership in public view structs unless there is a clear
   cleanup function.
3. Keep `UnitInfo` for `ListUnits()` separate from full property views.

Verification:

```sh
meson test -C build -v test-systemd1-broker
```

## Milestone 3: Read-Only Manager Method Helpers

Goal: implement method-level logic without a running daemon.

Files to change:

1. Extend `src/systemd1-broker/systemd1-broker-bus.c` with helpers for read-only
   Manager methods.
2. Extend focused D-Bus tests.

Tests first:

1. `ListUnits()` helper returns all registered units in insertion order.
2. `ListUnitsFiltered(as states)` returns units matching load state, active
   state, or sub state.
3. `ListUnitsByNames(as names)` returns requested units and represents missing
   units consistently with systemd's tuple shape for not-found entries, or
   returns a documented error if this first slice chooses fail-fast behavior.
4. `ListUnitsByPatterns(as states, as patterns)` filters by state and glob
   pattern.
5. `GetUnit(s name)` returns the existing unit path.
6. `GetUnit(s missing)` returns a systemd-compatible no-such-unit error mapping
   at the facade layer; helper-level tests may assert `-ENOENT`.
7. `LoadUnit(s name)` aliases `GetUnit()` for already indexed units in this
   milestone.

Implementation notes:

1. Keep pattern matching in helper code using existing systemd string/glob
   helpers already available in-tree.
2. Do not implement unit-file loading in `LoadUnit()` yet; document the behavior
   in the test name.
3. Use helper-level errno values and map them to D-Bus errors in Milestone 5.

Verification:

```sh
meson test -C build -v test-systemd1-broker
```

## Milestone 4: Job Operation Semantics

Goal: prepare the mutating Manager/Unit methods before wiring sd-bus handlers.

Files to change:

1. Extend `src/systemd1-broker/systemd1-broker.h` with operation helpers if the
   existing job helpers are not enough.
2. Extend `src/systemd1-broker/systemd1-broker.c`.
3. Extend focused tests.

Tests first:

1. `StartUnit(name, "replace")` creates a `start` job and attaches it to the
   Unit `Job` field.
2. `StopUnit(name, "replace")` creates a `stop` job.
3. `RestartUnit(name, "replace")` creates a `restart` job.
4. `ReloadUnit(name, "replace")` completes a broker-local reload job without
   invoking backend operations.
5. `TryRestartUnit()` returns no job for inactive units and creates a restart job
   for active units.
6. Unsupported modes return `-EOPNOTSUPP`.
7. Conflicting jobs return `-EBUSY` or a dedicated transaction error value that
   maps to `org.freedesktop.systemd1.TransactionIsDestructive`.
8. Completing a job updates final Unit state before the job disappears from
   `ListJobs()`.

Implementation notes:

1. This milestone can use a fake synchronous backend decision embedded in the
   state model: start maps to running, stop maps to stopped, restart maps to
   running.
2. Keep a TODO in the plan, not in code, for replacing this with a real adapter.
3. If a backend abstraction is introduced now, keep it minimal and in C:
   function pointers for query/start/stop/restart plus an opaque userdata.

Verification:

```sh
meson test -C build -v test-systemd1-broker
```

## Milestone 5: sd-bus Object Facade

Goal: expose the tested helpers through `org.freedesktop.systemd1` interfaces.

Files to change:

1. Add `src/systemd1-broker/systemd1-broker-dbus.h`.
2. Add `src/systemd1-broker/systemd1-broker-dbus.c`.
3. Update `src/systemd1-broker/meson.build`.
4. Add or extend D-Bus facade tests.

Tests first:

1. Register Manager, Unit, and Job vtables on an `sd_bus` object in test scope.
2. Assert introspection includes `org.freedesktop.systemd1.Manager`,
   `org.freedesktop.systemd1.Unit`, and `org.freedesktop.systemd1.Job` for
   objects that exist.
3. Invoke `ListUnits`, `GetUnit`, `LoadUnit`, and `ListJobs` through sd-bus and
   assert reply signatures and values.
4. Invoke `StartUnit`, complete the fake operation, and assert `JobNew`, Unit
   `PropertiesChanged`, and `JobRemoved` ordering once signal tests are practical.
5. Assert unsupported methods return deterministic D-Bus errors.

Implementation notes:

1. Use static vtables similar to `src/core/dbus-manager.c`, `src/core/dbus-unit.c`,
   and `src/core/dbus-job.c`, but only for the supported subset.
2. Map helper errors at the edge:
   `-EINVAL` to `org.freedesktop.systemd1.InvalidName` when name-related,
   `-ENOENT` to `org.freedesktop.systemd1.NoSuchUnit`, `-EOPNOTSUPP` to
   `org.freedesktop.DBus.Error.NotSupported`, and job conflicts to
   `org.freedesktop.systemd1.TransactionIsDestructive`.
3. Keep object lookup by path centralized so Unit and Job fallback vtables share
   the same escaping rules.

Verification:

```sh
meson test -C build -v test-systemd1-broker
```

## Milestone 6: Minimal Broker Executable

Goal: run a local broker process for manual and automated smoke tests.

Files to change:

1. Add `src/systemd1-broker/systemd1-broker-main.c`.
2. Update `src/systemd1-broker/meson.build` with an executable entry.
3. Add a manual or integration-style test only if it is reliable in this build
   environment.

Behavior:

1. Accept `--user` and `--system` modes, but allow a test-only `--socket=PATH`
   override to avoid touching `/run/systemd/private` during tests.
2. Seed the Manager with one or more static test units until a metadata indexer
   exists.
3. Listen on a Unix stream socket and speak D-Bus to clients through sd-bus server
   support.
4. Serve Manager, Unit, and Job objects through the facade from Milestone 5.
5. Exit cleanly on `SIGTERM`/`SIGINT`.

Smoke validation:

1. Start the broker on a temporary socket.
2. Use `busctl --address=unix:path=$SOCKET introspect org.freedesktop.systemd1 /org/freedesktop/systemd1` if `busctl` supports direct addresses in the environment.
3. If `systemctl` cannot target an arbitrary private socket without environment
   changes, defer full `systemctl` smoke tests until a controlled runtime dir
   harness is added.

Verification:

```sh
meson test -C build -v test-systemd1-broker
```

Do not run `meson compile` separately before tests; `meson test` rebuilds needed
targets.

## Execution Order

Execute one milestone at a time. For each milestone:

1. Add or update focused tests first and confirm the expected failure when
   practical.
2. Implement the smallest code change that passes the new tests.
3. Run `meson test -C build -v test-systemd1-broker`.
4. Run `git diff --check` after any non-trivial edit.
5. Stop and reassess if the implementation starts pulling in systemd core
   `Manager` internals, full unit-file parsing, or PID1/cgroup behavior.

## Completion Criteria

The Phase 1/2 slice is complete when:

1. Focused tests cover UnitInfo and JobInfo D-Bus tuple encoding.
2. Focused tests cover Unit and minimal Service property views.
3. Focused tests cover read-only Manager method behavior.
4. Focused tests cover start/stop/restart job lifecycle and completed job
   removal from `ListJobs()`.
5. The broker exposes Manager, Unit, and Job sd-bus vtables for the supported
   subset.
6. A minimal executable can serve at least one static unit over a test socket, or
   the exact blocker for doing so is documented in the spec.
7. `meson test -C build -v test-systemd1-broker` passes.
8. `git diff --check` reports no whitespace errors.
